Project Repository

Description

This is the project you are currently viewing. This Windows 95 themed web application uses the Angular framework.

Features Highlight:

Window Frame Component

The Window Frame component is arguably the most important component in this whole project. It is a wrapper I developed that enables me to painlessly make any HTML content act as a "window", similar to how program windows work on many operating systems. It enables the content to be viewed, moved, scaled, closed, minimized, etc. Every program in this web app is wrapped by a window-frame component.

Internet Explorer Component

Replicating Internet Explorer was the best and most efficient way to enable the "portfolio" aspect of this web application. Any project I wish to describe in detail can be given its own "website", which can be rendered within the Internet Explorer component. This keeps all portfolio content in one accessible location. The use of Angular Routing also makes each project's web page simple to share with others (try clicking on the "Copy" or "Mail" buttons above!).

Calculator Program Component

Inspired by Windows 95's calculator, I developed a math function library using C and compiling it to WebAssembly, and connected to the front-end of this web app using Angular and TypeScript. The calculator program can be accessed from the Start Menu under Programs > Accessories.

Routing Implementation

Routing was one part that lead me to choose Angular for this project. I knew I wanted to be able to access this web application and quickly load any programs from the URL bar, which routing enables. Each program opens when it is added as a parameter to the "programs" query, which in some cases can then be used to render components in certain ways (for example, system-monitor does not use any parameters, but internet-explorer does use parameters to render "websites").

Developer Tools

The following developer tools were utilized to enhance the developer experience:

  • Creating a System Monitor component to display relevant browser and code information within the web app. This program is accessible via the Start Menu.
  • Unit tests have been implemented for all relevant components (anything that has logic), and GitHub Actions have been configured for this project repository for testing and building.
  • Prettier was utilized to ensure consistent code formatting across the entire codebase.

Going Off the "Standard Path"

Not all tech used for this project is "essential" to its operation. This idea mainly focuses on Electron and Docker. While this website primarily serves as my portfolio, I also wanted it to serve as a "code sandbox" I could utilize to learn new technologies. This design philosophy allowed me to easily incorporate features such as Electron, Docker, and WebAssembly.

Why the Windows 95 theme and functionality?

First, I did not want to create what looked or felt like "just another website". I wanted to avoid blending in with other developers who create a simple multi-page website using React (not that there is anything wrong with that!). Instead, I wanted to stand out.

Second, I wanted the development of my website to challenge me to learn new skills, which was a goal I achieved. The creation of this website taught me new skills / improved my existing skills, such as Angular, HTML, SCSS / CSS, TypeScript & JavaScript, C, Web Assembly, programming logic, graphic design softwares, cross browser development, and much more. A simple multi page website can only teach the developer so much. Creating a web application that replicate the look and feel of an operating system can teach much more.
