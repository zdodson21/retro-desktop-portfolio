Description

This is the project you are currently viewing. This Windows 95 themed web application created using the Angular framework.

Features Highlight:

Window Frame Component

The Window Frame component is arguably the most important component in this whole project. It is a wrapper I developed that enables me painlessly make any HTML content act as a "window", similar to how program windows work on many operating systems. It enables the content to be viewed, moved, scaled, closed, minimized, etc. Every program in this web app is wrapped by a window-frame component.

Internet Explorer Component

Replicating Internet Explorer was the best and most efficient way to enable the "portfolio" aspect of this web application. Any project I wish to describe in detail can be given its own "website", which can be rendered within the Internet Explorer component. This keeps all portfolio content in one easily accessible location. The use of Angular Routing also makes each project's web page simple to share with others (try clicking on the "Mail" button above!).

Routing Implementation

Routing was one part that lead me to choose Angular for this project. I knew I wanted to be able to access this web application and quickly load any programs I wished to use with ease, which routing enables. Each program opens when it is added as a parameter to the "programs" query, which in some cases can then be used to render components in certain ways (for example, system-monitor does not use any parameters, but internet-explorer does use parameters to render "websites").

If you are confused, I suggest opening various programs from the Start menu and watching the web address in the URL bar. It will change based on what programs are open, and in certain cases (such as internet-explorer ) affect how the program is rendered as well. This behavior allows anyone to open programs from their browser's address bar without first rendering the web app. This feature will be more useful as more programs are created and added (such as a calculator or paint program).

Developer Tools

The following developer tools were utilized to enhance the developer experience:

  • Creating a System Monitor component to display relevant browser and code information within the web app. This program is accessible via the Start Menu.
  • Unit tests have been implemented for all relevant components (anything that has logic), and GitHub Actions have been configured for this project repository for testing and building.
  • Prettier was utilized to ensure consistent code formatting across the entire codebase.

Going Off the "Standard Path"

Not all tech used for this project is "essential" to its operation. This idea mainly focuses on Electron and Docker. While this website serves as my portfolio, I also wanted it to serve as a "playground" I could utilize to learn new technologies. This design philosophy allowed me to easily incorporate Electron packaging and Docker containerization.

A future feature that will follow this philosophy will be the calculator. Ideally, I would like to write the calculator program logic in C , then compile it to WebAssembly so it can be used by the calculator front-end (Angular component). This isn't the most practical or efficient way to complete the task or writing a calculator, but it will give me a start in learning the C programming language and gaining a better understanding of WebAssembly. If this route does not work, then I will resort to creating the calculator in TypeScript instead.

Story Time

Why did I create this web application? That is the question that has many answers. First, I didn't want to blend in with others by creating a website made up of a few pages, some text, pictures, and being "made with React". I wanted stand out, and creating a "standard website" seemed too simple, and would have been a project that wouldn't have benefitted me much as a developer. I preferred to make something that would challenge me to learn new things (such as a new framework with Angular, new languages with TypeScript and SCSS, new concepts such as unit testing, browser routing, Electron application bundling, Docker containerization, etc.).

Second, I originally thought of the general idea for this web application (themeing and functionality similar to Windows 95) in 2023, while I was still in college. Unfortunately for me, I was not the first to come up with the idea of creating a website with this theme, but that gave me the motivation to create the best version of a Windows 95 themed website I had ever used. I hope I have achieved this goal, but that is up for any users to decide.

Finally, I have always liked the visual style of Windows 95, which is part of the reason I chose to style this web application around that theme. But the idea of creating a web application that felt like using an operating system brought fourth numerous expected and unexpected challenges, all of which taught me new skills, programming techniques, and general aspects of software design. People tend to say that one of the best ways to learn software design is by copying something else (usually referring to something like Netflix), so replicating an operating system was sure to challenge me.
