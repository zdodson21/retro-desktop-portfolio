Calculator (Retro Desktop Portfolio Program)

Component Source Code Open Caluclator (will close all other programs)

Description

Developed using C compiled to WebAssembly alongside Angular, the calculator program is a faithful recreation of the calculator program from Windows 95. Angular is used to handle any logic on the frontend, then calls functions from a WebAssembly file to handle any mathematical logic.

Note: Some calculator logic is based on approximations (currently), so it should not be used for any calculations that require extreme precision.

Why C and WebAssembly? Why not Natively with Angular / TypeScript?

This is one of those projects I wanted to complete to learn new skills, not just to complete it in the most practical way. Because a calculator relies on backend logic, I figured it would be the best program to recreate to learn C, compiling, and WebAssembly. Angular and TypeScript are still used for both the frontend and ensuring any functions in WebAssembly can be called safely.

Is this the most practical way to create a calculator for the web? Probably not, Typescript would likely be plenty fast enough to handle the mathematical logic required, but it taught me a lot about programming in a lower-level language such as C, how compiling works, and provided a better understanding of how WebAssembly works. And technically it should be "faster" than handling the math logic with TypeScript, which is a (marginal) benefit.
