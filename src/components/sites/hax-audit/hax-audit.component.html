HAX Audit Open Source Contribution (February 2025 - March 2025)

Tech Used:

Pull Request

Description:

Created a read-only CLI tool to analyze web components created using the HAXTheWeb CLI tool . This tool will find any CSS within a web component and inform the user of any values that should be changed to meet the standards of the Design, Develop, Destroy! (DDD) design system .

Details:

  • .dddignore File Type
    • Created an interpreter for the .dddignore file type, enabling the audit tool to create a list of any files that should not be audited. This route was chosen over targeting JavaScript files to provide the user with more control and to future proof the auditing tool (such as adding TypeScript web component support in the future for example).
    • Developed a VSCode extension using the Yo Code Generator to natively support the .dddignore language syntax, providing highlighting for comments.
      • Extension Repository VSCode Marketplace Page
  • Specific suggestions are provided for numerous types of CSS values. Some examples are specific CSS colors such as "red" and "yellow", or "px" values. These types of values are detected, analyzed, and an applicable value is provided to the user. For example, if a user sets "color: red;", they will be informed that they should instead set it to "--ddd-theme-default-original87Pink", which is the closest color in the DDD design system. If no specific value can be provided, the developer will be instructed to look at the documentation.
  • A return code is provided for use in CI/CD pipelines, allowing automated systems to know whether or not a web component is compliant with the DDD design system.
